### 登录页 (index.html)
- 机构标识栏（医院名 + 系统版本）
- PHI 安全合规警告
- ID 001 患者卡片（含沈若楠离职状态）
- 评估员编号 + 授权码表单
- 沈若楠加密便签（3次失败后逐字显示动画）
- 任意非空凭据通过（demo 阶段）

### 主系统 (system.html)
- **顶部状态栏**：医院标识、评估员 badge、证据完备度进度条
- **左侧文件树**：4个区块（人格档案/治疗转录/外部证据/医疗记录），可折叠
- **中央阅读区**：支持 4 种文档类型渲染
  - profile（人格档案，含段落 + 关键摘录）
  - transcript（治疗转录，含说话者标注）
  - evidence（外部证据报告）
  - table（证据表格，含 highlightable 单元格）
- **右侧分析面板**：
  - 矛盾板（双槽钉选 + 比对按钮）
  - 证据链（P1-P8 节点 + 连接线）
  - 人格关系图谱（SVG 环形布局）
  - 提交按钮 + 结局模态

### 核心交互
- **高亮**：单击 `.highlightable` 文本选中/取消
- **钉选**：双击高亮文本 → 自动填入槽 A 或 B
- **比对**：双槽有内容时激活按钮 → 匹配 CONTRADICTION_RULES
- **自动解锁 P8**：P1-P7 全解锁时由引擎自动触发
- **假矛盾（noise）**：存在 3 条，触发时不产生任何效果
- **7 个结局**：A-G，通过提交模态选择结论触发

## 设计决策记录

1. **监控表的行结构** — 混合了普通数组行和含 highlightable 单元格的行（21:08 医院通行证），app.js 的 table 渲染器已正确处理两种格式
2. **矛盾 pair 排序** — 所有 pair 键按字母序排列，`doCompare` 取两 pin ID 排序后查找
3. **noise 静默处理** — 假矛盾的 title/detail 为 null，比对逻辑已正确处理不触发任何东西
4. **结局 E 隐藏选项** — 只在 P7 已解锁时在提交下拉菜单中显示
5. **P8 自动解锁** — 提交时的结局逻辑检查 `unlockedContradictions.includes('P8')`；但目前引擎并没有在 P1-P7 全解锁时自动添加 P8——需要在 `doCompare` 中添加这个检查
