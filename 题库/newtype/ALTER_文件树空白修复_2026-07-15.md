# ALTER 文件树空白 Bug 修复

**时间：** 2026-07-15 20:17  
**症状：** 进入 system.html 后左侧档案列表一片空白，没有任何文件。

## 根因

`buildFileTree()` 中 `createTreeSection()` 返回的对象结构：

```
{
  el: DOM元素（分组容器）,
  items: [],        // 数据数组
  add: function()   // 把 items 渲染成 DOM 节点
}
```

之前的代码只做了 `section.items.push(...)` 把数据塞进数组，然后直接 `tree.appendChild(section.el)` 把空容器挂到 DOM。**`.add()` 从来没被调用过**，所以四个分组的 HTML 子节点一次都没生成。

## 修复

给 `buildFileTree()` 中每个 section 在 `appendChild` 之前补上 `.add()` 调用：

```js
persSection.add();   // 之前缺
transSection.add();  // 之前缺
extSection.add();    // 之前缺
medSection.add();    // 之前缺
```

## 验证

- `node -c` 语法检查通过
- HTTP 200 返回的 app.js 包含 4 个 `.add()` 调用
- `buildFileTree()` 在初始化时被正常调用
